# User Workflow

## Requester workflow

1. User logs in.
2. User creates a ticket.
3. User checks ticket status.
4. User adds updates or replies to technician questions.

## Technician workflow

1. Technician reviews open tickets.
2. Technician assigns or starts work on a ticket.
3. Technician adds troubleshooting notes.
4. Technician links the ticket to an affected asset.
5. Technician resolves the ticket with a resolution summary.

## Manager workflow

1. Manager checks dashboard.
2. Manager reviews overdue and critical tickets.
3. Manager exports CSV reports.
4. Manager uses status data to understand workload.
